File security

On a Linux system, every file is owned by a user and a group user. There is also a third category of users, those that are not the user owner and don't belong to the group owning the file. For each category of users, read, write and execute permissions can be granted or denied. Use ls -l to show the file permissions

chmod:

chmod u+x,go-rwx

Users and privileges

Not every user has the same privileges. We have apache, jenkins, ... users. To run a command as a different user:

sudo -u apache -i
